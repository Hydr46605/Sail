# Sail 0.1.0-alpha.5

This alpha moves the Minecraft gateway beyond hygiene into the first real
limbo/hybrid implementation while keeping the release surface explicit.

## Changes

- Adds LimboAPI-backed `limbo` and `hybrid` gateway modes for Velocity.
- Keeps `kick` mode as the no-LimboAPI fallback and fails closed when a
  limbo-backed mode is selected without LimboAPI available.
- Adds gateway status lines for backend target, LimboAPI requirement and
  availability, and waiting limbo players.
- Preserves the auth challenge mode in registry status responses so operators
  can distinguish `kick`, `limbo`, and `hybrid` challenges.
- Clarifies Paper companion diagnostics: Paper can see forwarded Sail identity
  metadata, but cannot prove the gateway mode or firewall boundary by itself.
- Documents the LimboAPI dependency and backend target configuration for the
  alpha gateway.
