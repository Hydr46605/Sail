# Sail v0.1.0-alpha.7

Alpha pre-release. Not for production use.

## Highlights

- **Server registration and lifecycle management** — Register servers through the console, receive API keys and claim codes, and deregister servers when decommissioning.
- **Audit trail and signing key rotation** — View account audit events and manage registry signing keys (list, revoke) from the console.
- **Name lookup with claim flow** — Check name availability and claim unclaimed names directly from the dashboard.
- **Gateway connection gating** — New `registry-failure-policy` config option (`fail_closed` / `fail_open`) to control player access when the registry is unreachable.
- **Health monitoring dashboard** — Server heartbeat status (online/stale/offline), server cards with health indicators, and operator summary metrics.
- **Unified notification system** — Consistent error and success banners across the entire console UI.

## New features

### Registry API

- `POST /v1/servers` — Register a new server with an API key and claim code.
- `POST /v1/servers/claim` — Consume a claim code to retrieve the API key JWT.
- `POST /v1/servers/heartbeat` — Record a server heartbeat for health monitoring.
- `POST /v1/servers/:server_id/deregister` — Disable a server and revoke its API key.
- `GET /v1/names/:name` — Public name lookup returning claim status and premium name check.
- `GET /v1/console/audit-events` — List audit events for the authenticated account.
- `GET /v1/console/signing-keys` — List registry signing keys with lifecycle status.
- `POST /v1/console/signing-keys/:kid/revoke` — Revoke a signing key.

### Console

- Server registration form with API key delivery modal (copy, download, QR code).
- Server cards showing health status, heartbeat time, and deregister action with confirmation dialog.
- Name lookup component with check-and-claim flow.
- Operator summary panel with session, server, and reuse policy metrics.
- Audit log table with severity pills (info/warning/high/critical).
- Signing keys table with revoke action and confirmation dialog.
- Success banners for session revocation and server deregistration with auto-dismiss.
- Relative time-until-expiry on session rows.
- Theme switch (light/dark) with system preference detection.

### Gateway

- `registry-failure-policy` config field: `fail_closed` (default, deny login when registry is unreachable) or `fail_open` (allow login).
- API key authentication for server-to-registry communication.
- Server heartbeat integration.

### Companion

- Identity lookup renderer for `/sail lookup <player>`.
- Status renderer for `/sail status`.
- Backend diagnostics command.

## Infrastructure

- OpenAPI spec updated with all new endpoints, `Audit`, `Signing Keys`, and `Names` tags.
- API contract verification (`verify-api-contracts.mjs`) checks served routes against the OpenAPI spec.
- Unified `Banner` component replacing separate `ErrorBanner` and `SuccessBanner`.
- PostgreSQL migrations for server registration, API keys, signing key lifecycle, and audit events.
- Rate limiting on all registry endpoints.
